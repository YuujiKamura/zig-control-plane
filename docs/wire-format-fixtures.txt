# CP wire-format fixtures
#
# Format:
#   - Blocks are separated by a line containing only `===`.
#   - Each block contains:
#       # human-readable note (optional, may repeat)
#       >>> request line
#       <<< response (may use `\n` to encode multi-line responses)
#   - Escape conventions used in BOTH `>>>` and `<<<` payloads:
#       `\n`  -> single newline byte (0x0A)
#       `\s`  -> single space byte (0x20). Use this only at the END of a
#               line where text editors might strip trailing whitespace.
#       `\\`  -> literal backslash. (Reserved; not used in current fixtures.)
#   - Lines beginning with `#` outside a payload are notes; ignored.
#   - Blank lines inside a block are tolerated; they reset nothing.
#
# Mock provider state assumed by these fixtures (see
# vendor/zig-control-plane/src/main.zig MockState):
#   session = "test-session"
#   viewport buffer = "user@host:~$ "  (with trailing space, no newline)
#   tab_count = 2, active_tab = 0
#
# `apprt-only` in the note marks fixtures that exercise the apprt overlay
# (BUSY envelopes, `stale:` wire envelope). The Zig server-side test runs
# the fresh fixture against ControlPlane.handleRequest directly, and
# synthesises the apprt-only fixtures via the documented format helpers.
